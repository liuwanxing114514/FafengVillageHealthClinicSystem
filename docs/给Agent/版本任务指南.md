# Cursor 版本任务指南

本文档说明：**每个版本开新对话时，需要给 Cursor 什么材料、怎么下指令、怎么验收**。  
诊所机器上的升级操作见 [`../共用/DEPLOYMENT.md`](../共用/DEPLOYMENT.md)。给人看的白话说明见 [`../给人看/后续版本操作说明.md`](../给人看/后续版本操作说明.md)。版本定义见 [`../共用/ROADMAP.md`](../共用/ROADMAP.md)。

---

## 使用方式

1. **开新对话**（或新 Agent 会话），避免上下文过长混乱。
2. **用 @ 引用**下面「必读文件」中的文档和关键代码目录。
3. **复制对应版本的 Prompt**，填好 `【】` 中的实际信息。
4. 要求 Cursor **一次只做一个版本**，完成后对照验收清单。
5. 验收通过后再做下一版；诊所上线按 [`../共用/DEPLOYMENT.md`](../共用/DEPLOYMENT.md) 与 [`../给人看/后续版本操作说明.md`](../给人看/后续版本操作说明.md) 执行。

---

## 每次都要给 Cursor 的通用材料

无论哪一版，建议始终 @ 引用：

| 文件 | 路径 |
| --- | --- |
| 路线图 | `docs/共用/ROADMAP.md` |
| 开发交接 | `docs/给Agent/开发交接.md` |
| 业务大纲 | `docs/共用/大纲.md` |
| 部署升级 | `docs/共用/DEPLOYMENT.md` |
| 使用者需求 | `docs/给人看/使用者功能确认.md` |
| AI 架构 | `docs/给Agent/AI架构.md` |

若项目已有代码， additionally @：

| 路径 | 原因 |
| --- | --- |
| `docker-compose.yml` | 服务编排 |
| `.env.example` | 配置项约定 |
| `backend/` 相关模块包 | 避免重复造轮子 |
| `frontend/src/` 相关页面 | 保持 UI 一致 |
| 最近一次 Flyway 迁移脚本 | 数据库变更衔接 |

### 通用约束（可贴在每次 Prompt 末尾）

```text
必须遵守：
- 模块化单体，不做微服务（AI/OCR/Whisper 可独立容器，业务仍在一个 backend）。
- 库存变更必须走流水；禁止直接改库存数量。
- 出库关联患者和处方；库存不足阻止出库；出库批次由系统 FEFO 推荐，用户手动确认后才扣减（可改选）。
- AI/Agent 只能写 ai_draft 草稿，人工确认后才写正式数据。
- 核心数据软删除，保留审计。
- 金额/数量不用 float；Flyway 管理数据库变更。
- 一次只完成当前版本，不要顺带做下一版本功能。
- 完成后说明：改了哪些文件、如何验证、需要更新 `.env.example` 和 `docs/共用/DEPLOYMENT.md` 哪些项。
- 后端自测：用 **IDEA MCP**（`user-jetbrains` → `execute_terminal_command`）跑 `mvn test`；本机 Cursor 终端常无 `JAVA_HOME`，不要只跑 shell 就报「测不了」。
- 前端自测：`cd frontend && npm run build`。
- 测试通过后更新 `docs/给Agent/功能点与实现记录.md`，再 push / 合并对应分支到 `develop`（需我明确要求时再 commit）。
```

| 步骤 | 工具 | 说明 |
| --- | --- | --- |
| 前端构建 | Shell 或 IDEA 终端 | `cd frontend; npm run build` |
| 后端测试 | **user-jetbrains** → `execute_terminal_command` | 在 `backend` 目录执行 `mvn test`；若报 `JAVA_HOME` 未配置，设置 `$env:JAVA_HOME='C:\Users\lwx\.jdks\corretto-21.0.11'`（以 `.idea/misc.xml` 里 `project-jdk-name` 为准） |
