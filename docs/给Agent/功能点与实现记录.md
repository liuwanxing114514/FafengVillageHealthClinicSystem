### v2.0 待实现（ROADMAP 摘要）

- **受控工具**（只读或写 `ai_draft`）：`searchMedicine`、`queryInventory`、`queryExpiringMedicine`、`searchPatient`、`searchPatientVisit`、`generateOutboundDraft`
- **Agent 编排**：意图识别 → 工具调用 → 结果摘要（DeepSeek function calling 或等价方案）
- **AI 助手页**：展示计划、工具名、参数摘要、查询结果；改数据动作为「待确认卡片」
- **`agent_execution_log` 表**：记录工具名、参数摘要、耗时、结果
- **禁止**：直接 UPDATE 库存/病历、任意 SQL、未注册工具

### v2.0 前置与验收

- **前置**：v1.3 DeepSeek 可用（诊所 `.env` 已配 API Key 或本地 stub）
- **验收**：问「阿莫西林还有多少」「哪些快过期」得正确回答；说「帮某药出库」只出待确认清单、不扣库存

### v1.6 现场待办（非阻塞 v2.0 开发）

- 预印纸 `preprinted-fafeng` 坐标在实机微调（`sys_setting` / 模板 JSON）

## v1.6 功能点分解（已完成）

### A. 药品建议零售价

- [x] Flyway：`medicine.suggested_retail_price NUMERIC(12,2) DEFAULT 0`
- [x] 后端 Entity/DTO/VO/导入模板贯通
- [x] 前端药品编辑、Excel 导入列

### B. 处方模板（预印纸对齐）

- [x] `sys_setting` 默认模板 JSON（`default-a4` + `preprinted-fafeng`）
- [x] 双模板渲染；设置页切换；预印不打印费别/卡号/金额/调剂
- [x] 打印 VO：门诊病历号仅数字、日期拆分、科别全科

### C. 就诊收费与欠款

- [x] `clinic_visit.amount_due` / `amount_paid`
- [x] API 返回推荐应收、参考进货成本（基于关联处方）
- [x] 病历页收费区；患者 `totalArrears`；欠款横幅
- [x] 手改应收后处方变更弹窗确认更新（onActivated）

### D. 病历独立入口

- [x] 菜单「病历」；`GET /api/visits` 分页搜索（关键词、日期、欠款筛选）
- [x] `VisitListView`；新建不强制 patientId
- [x] 患者可搜索选择；**弹窗快速建患者**，不跳转、保留病历表单

### E. 保存 PDF（v1.6 补丁）

- [x] 处方打印页工具栏「保存 PDF」（与「打印」并列）
- [x] `html2pdf.js` 按需加载；文件名 `处方-{姓名}-{日期}-{id}.pdf`

### v1.6 不做

- 支付渠道、分期收银；预印纸费别/卡号/调剂/处方金额打印
- 病历正文单独 PDF 页（仅处方打印页支持）
- v2.0+ Agent / RAG
