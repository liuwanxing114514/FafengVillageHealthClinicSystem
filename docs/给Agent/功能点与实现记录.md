### 能否并行开两个 Agent 对话？

**可以并行开发，但合并必须有序**（先 v2.0.1 → `develop`，再 v2.0.2 rebase `develop` 后合并）。

| 对话 | 分支 | 主要改动的目录 |
| --- | --- | --- |
| A：v2.0.1 | `v2.0.1-engineering` | 全库 Entity/Service、`pom.xml`、`SecurityConfig`、springdoc |
| B：v2.0.2 | `v2.0.2-spring-ai` | `ai/`、`agent/`、`pom.xml`（Spring AI 依赖）、相关测试 |

**并行时建议分工（减少冲突）**：

- **v2.0.1 会话**：Lombok + springdoc + 注释；**暂时不要改** `ai/`、`agent/` 包（留给 v2.0.2 会话一并 Lombok + Spring AI）。
- **v2.0.2 会话**：仅 `ai/`、`agent/` + Spring AI 依赖；改到的类在本会话内补 Lombok。

若两会话都全库改 Lombok，并行也可以，但合并 v2.0.2 时几乎必然要在 `ai/`、`agent/`、`pom.xml` 上手动解冲突。

**更稳做法**：只并行「v2.0.2 等 v2.0.1 合并后再开」——仍算两个对话，只是时间上错开半天。
