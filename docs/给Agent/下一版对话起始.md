## 当前下一版：v2.4 RAG 运维与 Embedding 管理

**分支**：`git checkout v2.4-rag-ops`（基线 `develop`，已含 v2.3）

```text
请按 ROADMAP v2.4 与功能点与实现记录开发「RAG 运维与 Embedding 管理」。

## 基线 develop 已有（勿重做）
- v2.2：visit_embedding、VisitEmbeddingService、sync-full/sync-incremental、GET embeddings/status
- v2.3：POST embeddings/search-similar、VisitFormView 侧边 SimilarVisitPanel

## v2.4 目标
1. 系统设置页：展示 Embedding 状态（pendingCount、latestSyncedAt 等）
2. 一键全量/增量同步按钮，结果 toast，进行中防重复点击
3. CLINIC_EMBEDDING_ENABLED=false 时禁用并说明，不报错
4. 更新 DEPLOYMENT.md「首次启用 RAG 全量同步」SOP

## 约束
- 不改 v2.2 同步算法与向量模型
- 模块化单体；一次一版
- 完成后：mvn test + npm run build，更新功能点与实现记录，合并 develop

## 开发环境
- IDEA profile=dev；DB application-dev.yml（postgres/admin）；测试密码 Test1234
- Embedding：硅基流动 BAAI/bge-m3 或 CLINIC_EMBEDDING_ENABLED=false 测降级
- 后端测试优先 IDEA MCP 跑 mvn test
```
