# 前端壳层与多标签约定

> **用途**：后续 Agent 开发前端页面时必须阅读本文档。  
> **实现版本**：v1.0.1（2026-07-08）  
> **技术栈**：Vue 3 + Vue Router + Pinia + Element Plus

---

## 1. 布局结构

登录 / 首次设密（`/login`、`/setup`）为**全屏页**，不使用主布局。

登录后所有业务页嵌套在 **`MainLayout`** 中：

```text
┌─────────────┬──────────────────────────────────────────────┐
│ AppSidebar  │ AppTabs（可关闭） + 顶栏右侧（用户 / 退出）      │
│ 可收起菜单   ├──────────────────────────────────────────────┤
│             │ router-view + keep-alive（最多 10 页缓存）     │
└─────────────┴──────────────────────────────────────────────┘
```

**处方打印**（`/prescription/:id/print`）为 **`standalone`** 路由，不在主布局与标签栏中；从新窗口打开（`window.open`）。

## 2. 关键文件

| 路径 | 职责 |
| --- | --- |
| `frontend/src/layouts/MainLayout.vue` | 壳层容器、keep-alive、顶栏 |
| `frontend/src/layouts/AppSidebar.vue` | 左侧菜单（可收起） |
| `frontend/src/layouts/AppTabs.vue` | 顶部多标签 |
| `frontend/src/stores/tabs.ts` | 标签打开 / 关闭 / 标题 |
| `frontend/src/config/menu.ts` | 固定菜单项、`MAX_TABS=10` |
| `frontend/src/composables/useTabTitle.ts` | 详情页动态改标签标题 |
| `frontend/src/router/index.ts` | 嵌套路由；`afterEach` 同步标签 |

## 3. 路由规则

### 在 MainLayout 内（有标签）

- `/` 首页
- `/patient`、`/patient/:id`
- `/medicine`、`/medicine/:id`、`/medicine/import`
- `/visit/:id`
- `/prescription/new`、`/prescription/:id`
- `/inventory/*`、`/ai`、`/settings`

路由 `meta.title` 为默认标签名；详情页加载数据后用 **`useTabTitle`** 覆盖，例如 `患者：张三`。

### 不在 MainLayout 内

| 路由 | meta | 说明 |
| --- | --- | --- |
| `/login` | `public: true` | 登录 |
| `/setup` | `public: true` | 首次设密 |
| `/prescription/:id/print` | `standalone: true` | 打印专用新窗口 |

---

## 4. 新开 / 切换标签

- **左侧菜单点击** → `router.push` → `router.afterEach` → `tabsStore.syncFromRoute(to)`
- **页面内跳转**（如列表点进详情）→ 同上，按 `fullPath` 区分（`/patient/12` 与 `/patient/34` 为两个标签）
- **点击标签** → `router.push(tab.path)`
- **关闭标签** → `tabsStore.removeTab`；若关的是当前页则跳转到相邻标签

同一 `fullPath` 不会重复开标签。

---

## 5. 详情页动态标题

```typescript
import { computed } from 'vue'
import { useTabTitle } from '@/composables/useTabTitle'

useTabTitle(computed(() => form.name ? `患者：${form.name}` : '新建患者'))
```

在 `onMounted` 与数据变化时自动更新当前标签标题。

---

## 6. keep-alive

`MainLayout` 内：

```vue
<keep-alive :max="10">
  <component :is="Component" :key="route.fullPath" />
</keep-alive>
```

- 切换标签保留表单状态
- 最多缓存 10 个实例（与 `MAX_TABS` 一致）
- 关闭标签后缓存由 LRU 回收，无需手动销毁

## 7. 新功能页 Checklist（Agent 必查）

- [ ] 路由挂在 `MainLayout` 的 `children` 下，并设 `meta.title`
- [ ] **不要**加「返回首页」按钮
- [ ] **不要**在页面内复制顶栏导航（用左侧菜单）
- [ ] 列表 → 详情用 `router.push`，会自动新开标签
- [ ] 详情页用 `useTabTitle` 显示友好标题
- [ ] 打印 / 全屏类页面用 `standalone` 或新窗口，不进标签栏
- [ ] 新增固定入口时同步更新 `config/menu.ts`

---

## 8. 菜单扩展

在 `frontend/src/config/menu.ts` 的 `MENU_ITEMS` 增加一项，并在 `AppSidebar.vue` 的 `iconMap` 中注册对应 Element Plus 图标。

---

## 9. 退出登录

`authStore.logout()` 会调用 `tabsStore.reset()` 清空标签，避免下一用户看到上一用户的打开页。
