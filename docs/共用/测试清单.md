# 测试清单

本文档区分 **本地开发可测** 与 **回归验收**（含需 NAS / 真机 HTTPS 的项）。  
部署步骤见 [`DEPLOYMENT.md`](DEPLOYMENT.md)；版本功能定义见 [`ROADMAP.md`](ROADMAP.md)。

**图例**

| 标记 | 含义 |
| --- | --- |
| 🤖 | 自动化，本地可跑 |
| 💻 | 本地浏览器手工（IDEA dev + `npm run dev`） |
| 🐳 | 本地 Docker Compose（需本机 Docker Desktop） |
| 📱 | 手机真机 + 局域网 HTTPS |
| 🗄️ | 群晖 NAS 生产环境 |

## 一、本地能测什么（无 NAS 内网）

### 1.1 自动化（每次发版必跑）

| # | 项 | 命令 / 方式 | 通过标准 |
| --- | --- | --- | --- |
| L-A1 | 后端单元/集成测试 | IDEA MCP → `backend` 目录 `mvn test` | 全部 PASS |
| L-A2 | 前端生产构建 | `cd frontend && npm run build` | exit 0，无 TS 错误 |

> Windows 终端若无 `JAVA_HOME`，**必须用 IDEA MCP** 跑 `mvn test`，不要只依赖 Cursor 终端。

### 1.2 本地 IDEA dev + 前端 dev（推荐日常手工）

**前置**：PostgreSQL 本地可连（见 `application-dev.yml`）；IDEA Run `ClinicApplication`，profile=`dev`；`cd frontend && npm run dev` → `http://localhost:5173`。

| # | 模块 | 检查点 |
| --- | --- | --- |
| L-B1 | 登录 | 空库 `/setup` 设密；`/login` 登录；改密 |
| L-B2 | 药品 | 新增、编辑、搜索、条码 |
| L-B3 | 库存 | 手动入库、出库（FEFO 确认）、流水、临期/不足首页 |
| L-B4 | 患者/病历 | 建档、录入、历史列表、收费字段 |
| L-B5 | 处方 | 开方、打印预览、保存 PDF |
| L-B6 | 扫码 | 入库/出库页条码输入（扫码枪模拟键盘） |
| L-B7 | Excel | 药品模板下载、导入预览 |
| L-B8 | 快捷语 | 候选、管理 CRUD |
| L-B9 | v2.6 | 库存流水导出 Excel；病历保存 PDF |
| L-B10 | AI 全关 | `.env`/Run Config 不设 API Key；各页无 500；无 Whisper/OCR 时入口友好降级 |

### 1.3 本地 Docker Compose（可选，验证容器化）

**前置**：本机 Docker Desktop；项目根 `cp .env.example .env`，`CLINIC_DATA_DIR=./clinic-data`。

| # | 项 | 命令 |
| --- | --- | --- |
| L-C1 | Core 三容器 | `docker compose up -d --build` |
| L-C2 | 首次设密 | 浏览器 `:8088`（或 `.env` 的 `FRONTEND_PORT`）→ `/setup` |
| L-C3 | 演示数据 | Windows：`.\scripts\seed-demo.ps1`；Git Bash/WSL：`./scripts/seed-demo.sh` |
| L-C4 | 备份脚本 | `.\scripts\backup.ps1` 或 `./scripts/backup.sh` 产出 zip/tar.gz |
| L-C5 | OCR profile | `docker compose --profile ocr up -d --build` + `CLINIC_OCR_URL=http://ocr-service:8000` |

**本地一般不测**：Whisper 容器（生产也不部署；用手机输入法语音）。

### 1.4 需 API Key 的 AI（本地可选）

在 IDEA Run Configuration 加载 `.env` 或手动填环境变量后重启 backend：

| # | 功能 | 条件 |
| --- | --- | --- |
| L-D1 | DeepSeek 整理草稿 | `CLINIC_AI_ENABLED=true` + Key |
| L-D2 | Agent 查询 | 同上 |
| L-D3 | RAG 全量同步 + 相似 Top-3 | `CLINIC_EMBEDDING_ENABLED=true` + Key；先 `seed-demo` 再设置页全量同步 |
| L-D4 | OCR 入库 | OCR 容器 + Chat 可用 |

### 1.5 本地难以完整覆盖（可跳过或延后）

| 项 | 原因 | 替代 |
| --- | --- | --- |
| DSM 任务计划备份 | 无 NAS | 本地手动跑 backup 脚本即可 |
| NAS 版本更新 SOP | 无第二套环境 | 文档走读 + 本地 git checkout 模拟 |
| 手机扫码（摄像头） | 须 HTTPS 安全上下文 | `npm run dev` HTTPS + 手机同 WiFi（见 L-E1） |
| 手机输入法语音 | 须手机 + HTTPS | 真机测一次即可 |
| PWA「添加到主屏幕」 | 生产 HTTPS 域名 | Chrome DevTools 设备模式测抽屉菜单 |

### 1.6 本地手机真机（可选，同一 WiFi）

```text
cd frontend && npm run dev   # 已启用 HTTPS
手机访问 https://<电脑IP>:5173，信任自签名证书
```

| # | 项 |
| --- | --- |
| L-E1 | 侧栏抽屉（≤768px） |
| L-E2 | 病历页输入法语音录入 |
| L-E3 | 入库/出库摄像头扫码 |

## 五、测试会话起始提示词（复制即用）

开**新对话**做本地/回归测试时，整段复制下面内容，把 `【】` 换成你的实际情况：

```text
@docs/共用/测试清单.md
@docs/共用/DEPLOYMENT.md
@docs/给Agent/功能点与实现记录.md

请按「测试清单」带我完成发版回归，不要改业务代码（除非测出 bug 需修复）。

## 环境
- 无群晖 NAS 内网，仅在本地 Windows 开发机测试
- 后端：IDEA profile=dev，PostgreSQL localhost（application-dev.yml）
- 前端：npm run dev → http://localhost:5173（手机真机可选：https://<电脑IP>:5173）
- 测试密码：Test1234
- Whisper：不部署；语音用手机输入法（不测 whisper 容器）
- AI API：【未配置 / 已配置硅基流动 Key，测 Agent+RAG】

## 范围
- 必做：测试清单 L-A1（IDEA MCP mvn test）+ L-A2（npm run build）
- 手工：L-B1～L-B10（v1.0 基线 + v2.6 导出/PDF）
- 回归：测试清单第二节 v1.0～v3.0 中标记 💻/🤖 的项
- 跳过：🗄️ NAS、📱 手机（除非我明确说「现在测手机」）

## 工作方式
1. 先跑自动化，贴结果；失败则修到全绿再继续
2. 再按模块逐项手工验收，每完成一项告诉我打勾内容
3. 发现问题只修最小 diff，不顺带重构
4. 全部通过后更新功能点与实现记录（如有需要）并汇总未测项（NAS/手机延后项）
```

**只跑自动化时**，可简化为：

```text
@docs/共用/测试清单.md

请用 IDEA MCP 跑 backend 的 mvn test，并 cd frontend && npm run build。
按测试清单 L-A1/L-A2 汇报结果；失败则修到全绿。
```
