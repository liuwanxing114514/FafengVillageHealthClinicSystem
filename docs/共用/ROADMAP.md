# 项目路线图

本文档是**版本划分的唯一权威来源**。`大纲.md`、`给Agent/开发交接.md` 中的阶段描述均应对齐本文档。

原则：

- 第一版（v0.1–v1.0）让诊所在**不依赖 AI** 的情况下完成日常业务。
- 后续版本逐步叠加 AI 能力；每版只做一个主题，避免在第一版过度实现。
- 开发过程中需为后续版本**预留接口和表结构**，但不必提前写完整实现。

## 第一期：基础业务 MVP（v0.1–v1.0）

目标：诊所日常可用——药品建档、入出库、库存预警、患者病历、处方打印、访问保护、备份恢复。

**v1.0 之前不得依赖 DeepSeek、OCR、Whisper、Agent。**

---

### v0.1 项目骨架

**前置依赖**：无

**目标**：

- 创建后端、前端、数据库、Docker Compose 基础结构。
- 建立统一代码规范和目录结构。

**交付**：

- Spring Boot 后端可启动。
- Vue 前端可启动。
- PostgreSQL 可连接（**使用 `pgvector/pgvector:pg16` 镜像**）。
- Flyway 可执行；**首条迁移 `V0` 启用 `vector`、`pg_trgm` 扩展**。

**需要预留**（本版只做目录/配置占位）：

- `.env.example` 中 AI 相关配置项注释。
- `backend` 中 `ai`、`agent` 空包或占位包。
- `db/migration/V0__enable_extensions.sql`（扩展一次装齐，v2.2 不再装插件）。

---

### v0.2 基础模块

**前置依赖**：v0.1

**目标**：

- 完成首次设置密码、登录、修改密码。
- 完成统一返回、异常处理、参数校验、审计日志基础能力。

**交付**：

- 可进入系统。
- 后台管理员可修改密码。
- 关键错误提示清楚。

**不做**：

- 多用户角色体系。

---

### v0.3 药品管理

**前置依赖**：v0.2

**目标**：

- 完成药品资料、单位换算、条码维护。

**交付**：

- 药品可新增、编辑、停用、搜索（名称、拼音首字母、条码）。
- 支持进货单价、默认库存下限（**按基本单位存储**，UI 可显示约等于几盒；默认 5 盒等价量，可按药品单独设置）。
- 支持整数单位换算（如 1 盒 = 12 片、1 瓶 = 100 颗）。
- 支持同一药品多规格。
- 液体药品容量只写在规格里，不按半瓶/毫升拆零。

**不做**：

- Excel 导入（v0.8）。
- 扫码匹配（v0.7）。

---

### v0.4 患者和门诊病历

**前置依赖**：v0.2

**目标**：

- 完成患者资料和病历录入。

**交付**：

- 患者可新增、编辑、搜索（姓名、电话、身份证号）。
- 患者字段：姓名、性别、**身份证号**、**出生日期**（可选）、年龄、电话、住址、备注。
- **年龄规则**：有身份证号则从证号解析出生日期并推算年龄；无身份证号但有出生日期则从出生日期推算；都没有则手填年龄。
- 病历字段：主诉、现病史、**既往史**、体温、血压、血氧、呼末二氧化碳、心率、脉象、过敏史、诊断、处理意见、备注。
- 可查看患者历史病历。

**不做**：

- 语音输入（v1.1）。
- AI 结构化整理（v1.3）。

---

### v0.5 处方和打印

**前置依赖**：v0.3、v0.4

**目标**：

- 完成处方记录和 A4 打印。

**交付**：

- 处方关联患者和病历。
- 处方明细关联药品，包含剂型、名称、规格、数量、单位、用法。
- 打印页第一行显示「发凤村卫生室处方签」。
- 打印内容包含患者姓名、性别、年龄、住址、电话、诊断、药品明细、日期、医生签名位置。
- 可根据处方生成**待出库清单**（手动确认后才扣库存，不在本版自动扣减）。

**不做**：

- 真实处方纸版式对齐（v1.6，需拿到纸质样式后）。
- Agent 自动生成出库清单（v2.0）。

---

### v0.6 库存管理

**前置依赖**：v0.3、v0.5

**目标**：

- 完成手动入库、手动出库、库存批次、库存流水、库存预警、盘点修正。

**交付**：

- 手动入库：填写药品、数量、单位、批号、有效期、进货单价、供应商 → 增加库存并生成流水。
- 手动出库：关联患者和处方 → 扣减库存并生成流水；库存不足时**阻止出库并提示**。
- 出库批号：**系统按 FEFO（先到期先出）推荐默认批次，用户手动确认后才扣减**；可改选其他批次；不得未经确认自动扣批。
- 盘点修正：必须填写原因，生成流水。
- 临期提前 3 个月提醒；库存不足按药品阈值提醒。
- 首页展示库存不足和临期药品。

**不做**：

- 扫码入出库（v0.7）。
- 批量出库（v1.5）。
- OCR/AI 入库（v1.4）。

---

### v0.7 扫码和手机扫码

**前置依赖**：v0.3、v0.6

**目标**：

- 支持扫码枪和手机浏览器扫码入出库。

**交付**：

- 扫码枪：条码作为键盘输入，自动匹配药品。
- 手机扫码：浏览器调用摄像头 + ZXing 识别药盒条码，提交后端匹配。
- 匹配成功后弹出数量、批号、有效期输入框；未匹配时提示先维护药品资料。
- 条码识别**不使用 OCR**。

**不做**：

- 拍进货清单识别（v1.4）。

---

### v0.8 Excel 模板导入

**前置依赖**：v0.3

**目标**：

- 系统提供固定格式 Excel 模板，用户按模板填写后导入药品或初始库存。

**交付**：

- 下载模板。
- 上传并解析。
- 预览导入结果，显示错误行。
- 用户确认后才正式写入。
- **不适配任意表头**，只认系统固定模板。

**模板字段**（与大纲 5.2.4 一致）：

药品名称、通用名、剂型、规格、基本单位、包装单位、换算关系、生产厂家、条码、进货单价、库存下限、批号、有效期、初始库存数量、备注。

**不做**：

- 智能识别用户自定义 Excel 格式。

---

### v0.9 AI 接口预留

**前置依赖**：v0.1

**目标**：

- 建立可拔插 AI 架构骨架，第一版业务不调用真实 AI。

**交付**：

- `AiProvider` 接口。
- `NoopAiProvider` 默认实现（AI 关闭时系统正常运行）。
- `DeepSeekAiProvider`、`LocalAiProvider` 空实现或占位类。
- AI 配置类（`.env` / `application.yml`）。
- `ai_draft` 表：存储待确认草稿（类型、内容 JSON、状态、来源）。
- 前端 AI 助手入口占位页（显示「功能开发中」即可）。

**不做**：

- 真实 DeepSeek API 调用（v1.3）。
- OCR、Whisper 服务（v1.1、v1.4）。

**需要预留的表/字段**：

- `ai_draft`：`draft_type`（`INBOUND` / `VISIT` / `OUTBOUND` / `QUERY`）、`status`（`PENDING` / `APPROVED` / `REJECTED`）、`payload`（JSONB）。

## 第二期：AI 辅助增强（v1.1–v1.3）

目标：减轻录入负担，所有 AI 输出仍为**草稿**，人工确认后才写入正式数据。

**共同约束**：

- 调用外部 API 前必须脱敏（规则见「待确认事项」）。

**目标**：

**交付**：

- 前端录音 → 上传音频 → Whisper 服务转写 → 填入当前字段。

**前置依赖**：v1.0

**目标**：

- 按字段提供常用文本候选，减少重复输入。

**交付**：

- 快捷语管理页：新增、编辑、删除。

**前置依赖**：v0.9、v1.1（可选，语音转写后可整理）

**目标**：

**交付**：

**不做**：

- OCR 图片识别（v1.4，本版只处理已有文本）。

**前置依赖**：v1.3、v0.6、v0.7

## 第四期：业务扩展（v1.5–v1.6）

### v1.5 批量出库

**前置依赖**：v0.6

**目标**：

- 一次选择多种药品批量扣减库存。

**适用场景**：

- 盘点损耗、集中领用、过期集中处理。

**交付**：

- 批量选择药品和数量。
- 必须填写统一原因。
- 每条药品独立生成出库流水。
- 关键操作二次确认。

**不做**：

- Agent 自动批量出库（v2.1 仍须人工确认）。

---

### v1.6 处方模板与就诊收费

**前置依赖**：v0.3、v0.4、v0.5

**目标**：

- 按真实预印处方纸对齐浏览器打印；病历记录应收/实收并汇总欠款；病历独立入口与快速建档。

**交付**：

**药品**

- 药品资料增加 **建议零售价**（`suggested_retail_price`，`numeric(12,2)`）。

**处方打印（预印纸对齐）**

- 保留现有 A4 通用模板为默认（`default-a4`）。
- 新增预印纸叠加模板（`preprinted-fafeng`）：仅打印数据，不重复打印纸上已有标签/下划线/标题。
- 可配置页眉（A4 用）、字段位置、字号、药品区行高；设置页可切换模板。
- 预印模式：日期拆年/月/日；门诊病历号位置只印**数字**（无英文标签）；科别固定「全科」。
- **完全不打印**：费别、卡号、处方金额、调剂。

**就诊收费与欠款**

- 每条就诊病历增加 **应收**（`amount_due`）、**实收**（`amount_paid`），`numeric(12,2)`。
- **默认应收** = 关联处方药品 Σ(建议零售价 × 数量)，打开病历时自动填入，旁注可手工修改。
- **参考进货成本** = Σ(进货单价 × 数量)，仅在病历界面展示，**不打印**。
- 医生手改应收后，若处方变更，**弹窗确认**是否按处方更新应收。
- 患者累计欠款 = 有效病历中 `max(0, 应收 - 实收)` 之和；进入患者详情或新建病历时醒目提示。

**病历独立入口**

- 左侧菜单「病历」；全量就诊列表（时间倒序）；支持关键词、日期、欠款筛选。
- 可直接新建病历：搜索选择患者；患者不存在时 **弹窗快速建档**，不离开病历页、不丢失已填内容。

**触发条件**：

- 处方对齐：需使用者提供纸质处方样本（已提供协会预印笺样本）。

**不做**：

- 支付渠道、分期、独立收银台。
- 预印纸打印费别/卡号/处方金额/调剂/进货成本。

## 第五期：AI Agent（v2.0–v2.1）

目标：可展示的工具调用闭环，Agent **只读查询 + 生成待确认动作**，不直接改库。

---

### v2.0 自然语言查询

**前置依赖**：v1.3、v0.6

**目标**：

- 文本或语音提问，Agent 调用受控工具返回答案。

**允许的工具**（只读或生成草稿）：

| 工具 | 说明 |
| --- | --- |
| `searchMedicine` | 按名称/条码查药品 |
| `queryInventory` | 查库存数量、批次 |
| `queryExpiringMedicine` | 查临期药品 |
| `searchPatient` | 查患者 |
| `searchPatientVisit` | 查历史病历 |
| `generateOutboundDraft` | 根据处方生成待确认出库清单（写入 `ai_draft`） |

**交付**：

- AI 助手页：展示 Agent 计划、调用的工具、结果摘要。
- 会改变数据的动作以「待确认卡片」展示，用户确认后才执行。
- Agent 执行日志（工具名、参数摘要、耗时、结果）。

**不做**：

- Agent 直接 `UPDATE` 库存或保存病历。
- 无限制 SQL 或任意 API 调用。

---

### v2.0.1 工程化（Lombok + API 文档 + 注释）

**前置依赖**：v2.0

**目标**：全库统一工程规范，**不改变业务行为**；便于开发与调试。

**交付**：

**Lombok（全库统一）**

- 根 `pom.xml` 引入 Lombok；`lombok.config` 统一配置。
- **Entity / Model**（MyBatis、Jackson 草稿类）：`@Data` + `@NoArgsConstructor`（替代手写 get/set）。
- **Service / Component**：`@RequiredArgsConstructor`（替代手写构造器）。
- **DTO / VO**：继续用 Java `record`（已是简洁形态，不强行改 Lombok）。

**springdoc OpenAPI（仅 dev 生效）**

- 依赖 `springdoc-openapi-starter-webmvc-ui`（Spring Boot 3 用 springdoc，不用旧版 springfox）。
- `application-dev.yml`：`springdoc.swagger-ui.enabled=true`；`prod` / `docker` profile **完全禁用** API 文档与 Swagger UI。
- `SecurityConfig`：仅 dev profile 放行 `/swagger-ui/**`、`/v3/api-docs/**`。
- 全局 `OpenAPI` Bean；主要 Controller 加 `@Tag` / `@Operation`（至少 AI、Agent、库存、处方模块）。

**注释**

- 各业务包 `package-info.java` 补模块说明。
- Controller、核心 Service、AI/Agent 编排类：类级 JavaDoc（业务规则、草稿门禁、禁止直接改库存等）。
- 不对每个 getter 或 trivial 方法写注释。

**不做**：

### v2.1 流程自动化

**前置依赖**：v2.0.2（建议）、v2.0、v0.5、v0.6

**目标**：

- 串联「开处方 → 生成待扣库存清单 → 医生确认 → 扣减库存 → 打印处方」。

**交付**：

- 处方保存后可一键生成待出库草稿。
- 医生在待确认页核对药品、数量、批次 → 确认后批量出库。
- 出库完成后跳转打印处方。

**约束**：

- 库存扣减必须有明确确认步骤。
- 任一步失败可回滚或保留草稿状态，不半扣库存。
