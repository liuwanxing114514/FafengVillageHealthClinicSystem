## 2. PostgreSQL 扩展策略（已确认）

### 原则

**所有需要的扩展在 v0.1 一次性启用**，避免生产库后期 `CREATE EXTENSION` 权限、镜像不一致、迁移顺序问题。

### Docker 镜像

```yaml
# docker-compose.yml
postgres:
  image: pgvector/pgvector:pg16   # 不要用普通 postgres:16
```

### Flyway 首条迁移 `V0__enable_extensions.sql`

```sql
-- v0.1 启用扩展（幂等，可重复执行语义由 IF NOT EXISTS 保证）
CREATE EXTENSION IF NOT EXISTS vector;      -- pgvector，v2.2 RAG 向量检索
CREATE EXTENSION IF NOT EXISTS pg_trgm;     -- 模糊搜索，v0.3 药品/患者名称搜索
```

| 扩展 | 启用版本 | 使用版本 | 用途 |
| --- | --- | --- | --- |
| `vector` | **v0.1** | v2.2+ | 病历向量、`visit_embedding` 表 |
| `pg_trgm` | **v0.1** | v0.3+ | 药品名、患者姓名模糊/拼音辅助搜索 |

### 验收（扩展）

```sql
SELECT extname, extversion FROM pg_extension WHERE extname IN ('vector', 'pg_trgm');
```

两条记录均存在即通过。

### v2.2 时不再做

- 不换 Postgres 镜像
- 不新增 `CREATE EXTENSION vector`
- 直接建 `visit_embedding` 表并使用 `vector(n)` 列

## 6. 数据库表与 Flyway 版本规划

| Flyway 版本 | ROADMAP | 内容 |
| --- | --- | --- |
| **V0** | v0.1 | `vector`、`pg_trgm` 扩展 |
| V1 | v0.2 | sys_user, sys_setting, audit_log |
| V2 | v0.3 | medicine, medicine_unit_conversion, medicine_barcode |
| V3 | v0.4 | patient, clinic_visit |
| V4 | v0.5 | prescription, prescription_item |
| V5 | v0.6 | inventory_batch, inventory_flow |
| V6 | v0.9 | ai_draft |
| V7 | v2.2 | visit_embedding（vector 列） |

### 6.1 patient（v0.4）

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | bigserial PK | |
| name | varchar(64) | |
| gender | varchar(8) | |
| id_card | varchar(18) | **必填**，唯一索引 |
| birth_date | date | 可从证号解析 |
| age | int | 展示用 |
| age_manual | boolean | true=手填，false=系统算 |
| phone | varchar(20) | |
| address | varchar(256) | |
| remark | text | |
| status | varchar(16) | ACTIVE/INACTIVE |
| created_at / updated_at | timestamptz | |

### 6.2 clinic_visit（v0.4）

含 `past_history`（既往史）、生命体征字段、主诉、现病史、过敏史、诊断、处理意见等。

### 6.3 medicine（v0.3）

`stock_threshold` 按**基本单位**存储；`purchase_price` numeric(12,2)。

### 6.4 inventory_flow（v0.6）

出库须关联 patient_id、prescription_id；记录 batch_id、变更前后数量。

### 6.5 ai_draft（v0.9）

`draft_type`、`status`、`payload` JSONB。

### 外键策略

**逻辑外键**（应用层 + 索引），不使用 DB 级 FOREIGN KEY。

## 7. 关键业务规则（设计约束）

- 出库：FEFO 推荐批次 → **用户确认**后扣减；库存不足**阻止**
- 年龄：证号 → 出生日期 → 手填
- AI：仅 ai_draft；T1 脱敏见 `AI架构.md`
- 软删除：status 或 deleted_at + 审计
