# 后续版本操作说明

给**您和诊所使用者**看的白话指南：第一期（v1.0）用稳之后，第二、三、四…期怎么上、谁做什么、要注意什么。

技术命令细节见 [`../共用/DEPLOYMENT.md`](../共用/DEPLOYMENT.md)。  
版本定义见 [`../共用/ROADMAP.md`](../共用/ROADMAP.md)。

---

## 一、先理解两件事

### 1. 升级不会丢数据

- **程序**在 `D:\clinic\`（可以整包换新）
- **数据**在 `D:\clinic-data\`（库存、病历、患者一直在）

升版本 = 换程序 + 自动升级数据库结构，**不会**清空已有病历和库存。

### 2. 每一期都可以「不用新功能」

| 阶段 | 不用 AI 时 |
| --- | --- |
| 第二期 AI（v1.1–v1.3） | 关掉 AI 开关，和以前一样手动录入 |
| 第三期 OCR（v1.4） | 不用拍照入库，继续手动/扫码入库 |
| 第五期 Agent（v2.0+） | 不用助手页，照常菜单里操作 |

## 二、谁做什么（一张表）

| 步骤 | 谁做 | 做什么 |
| --- | --- | --- |
| 1. 决定是否升级 | **您** | 看下面「版本时间表」，确认需要这版功能 |
| 2. 备份 | **您** | 升级前手动备份一次（见第四节） |
| 3. 写代码 | **Cursor** | 开新对话，按 [`../给Agent/版本任务指南.md`](../给Agent/版本任务指南.md) 开发**一个版本** |
| 4. 装到诊所电脑 | **您** | 复制新程序、`docker compose up`（见第四节） |
| 5. 验收 | **您 + 医生** | 按下面每期的「验收」打勾 |
| 6. 教医生用 | **您** | 新按钮/新流程口头或演示一遍 |

**医生日常不用管升级**，只需要知道新功能怎么用。

## 三、推荐节奏（别着急一次全上）

```text
v1.0 上线 ──→ 稳定用 2～4 周 ──→ 再考虑下一版
                │
                ├─ 想少打字 → v1.2 快捷语（最简单）
                ├─ 想口述病历 → v1.1 语音（要装 Whisper）
                ├─ 想 AI 整理 → v1.3 DeepSeek（要 API Key）
                ├─ 进货单拍照 → v1.4 OCR（要装 OCR 服务）
                └─ 想说话查库存 → v2.0 Agent（建议 v1.3 用顺了再上）
```

**原则：上一版用顺、备份做过、验收过，再升下一版。**

## 四、每次升级标准动作（您本人）

不管升哪一版，流程相同：

### 升级前（约 5 分钟）

1. 选诊所**不太忙**的时间（如下班后）。
2. 打开 PowerShell，执行备份：
   ```powershell
   cd D:\clinic
   .\scripts\backup.ps1
   ```
3. 记下备份文件名（万一要回滚）。

### 升级中（约 10～20 分钟）

4. 停服务：`docker compose down`
5. 把开发机上的**新程序**复制到 `D:\clinic\`（覆盖旧代码，**不要删** `D:\clinic-data\`）
6. 打开新的 `.env.example`，把**新增的配置项**抄进现有 `.env`（不要整个覆盖 `.env`）
7. 若说明里要求启用新容器（如 Whisper、OCR），按 [`DEPLOYMENT.md`](../共用/DEPLOYMENT.md) 第六节取消注释
8. 启动：`docker compose up -d --build`
9. 等 1～2 分钟，浏览器打开系统看能否登录

### 升级后（约 10 分钟）

10. 先测**老功能**：登录、入库、出库、病历、处方打印（必须都正常）
11. 再测**新功能**：看下面各期验收项
12. 告诉医生多了什么、要不要用

### 出问题怎么办

- 登录不了 / 库存不对 → 按 [`DEPLOYMENT.md`](../共用/DEPLOYMENT.md) **第七节回滚**
- 只是新功能不好用、老功能正常 → 可先关掉新功能（关 AI 开关 / 不点新按钮），不必立刻回滚

## 五、分阶段说明（升完每一期您会看到什么）

### 第一期 recap：v1.0（基础版，必须先有）

**您已有：** 药品、库存、患者、病历、处方打印、密码、每日备份。

**验收：** 断网也能正常开药、入库。

---

### 第二期：AI 辅助（v1.1 → v1.2 → v1.3）

可以**分开上**，不必一次全装。

#### v1.1 语音输入

| 项目 | 说明 |
| --- | --- |
| **您要多做** | 诊所机多跑一个 Whisper 容器；磁盘略增 |
| **医生看到** | 病历字段旁有「🎤 语音」按钮 |
| **怎么用** | 点语音 → 说话 → 文字出现在框里 → **自己改对再保存** |
| **重要** | 语音不会自动保存病历 |
| **不想用** | 不点语音按钮即可，和 v1.0 一样 |

**验收：** 录一段主诉，转写大致正确；关掉 Whisper 容器后手动录入仍正常。

---

#### v1.2 快捷语

| 项目 | 说明 |
| --- | --- |
| **您要多做** | 只换程序，一般不用新容器 |
| **医生看到** | 录入时出现常用语候选，点一下填入 |
| **管理** | 系统设置里可增删快捷语 |
| **依赖** | 不依赖外网、不依赖 AI |

**验收：** 主诉/诊断能点选常用语；不用 AI 也能用。

---

#### v1.3 DeepSeek 智能整理

| 项目 | 说明 |
| --- | --- |
| **您要多做** | 在 `.env` 填写 DeepSeek API Key；设 `CLINIC_AI_ENABLED=true` |
| **医生看到** | 可把一段话「整理成各字段草稿」，逐条确认 |
| **隐私** | 发出去前系统会自动打星号（姓、手机、门牌、身份证） |
| **重要** | 草稿不确认就不会写入正式病历 |
| **不想用** | `.env` 设 `CLINIC_AI_PROVIDER=noop` 或 `ENABLED=false` |

**验收：** 整理出草稿 → 不点确认 → 病历列表无新增；确认后才出现。

**费用：** DeepSeek 按调用量计费，可观察一段时间再决定是否长期开。

---

### 第三期：OCR 拍照入库（v1.4）

| 项目 | 说明 |
| --- | --- |
| **您要多做** | 多跑 OCR 容器；需 v1.3（DeepSeek 整理表格） |
| **医生看到** | 入库页可「拍照/上传进货单」 |
| **流程** | 拍**打印版**清单 → 系统出表格 → **逐条改对** → 点「批准入库」 |
| **不支持** | 手写单；不会自动入库 |
| **不想用** | 继续手动/扫码入库 |

**验收：** 拍一张清晰打印单 → 表格大致对 → 批准后才增加库存。

---

### 第四期：业务扩展（v1.5 / v1.6）

#### v1.5 批量出库

- **场景：** 盘点损耗、一批药过期一起处理  
- **流程：** 选多种药 → 填原因 → 二次确认 → 出库  
- **验收：** 每种药都有独立出库记录  

#### v1.6 处方纸对齐打印

- **前提：** 您提供**真实处方纸**扫描图  
- **再做：** 让 Cursor 按样本调打印位置  
- **验收：** 打印内容与处方纸对齐  

---

### 第五期：AI 助手（v2.0 → v2.1）

#### v2.0 说话查系统

| 项目 | 说明 |
| --- | --- |
| **医生看到** | 首页「AI 助手」，可问「阿莫西林还有多少」「哪些快过期」 |
| **重要** | 只**查**和**建议**；说「帮我把某药出库」只会出**待确认清单**，不会自己扣库存 |
| **依赖** | 建议 v1.3 已开且用顺 |

**验收：** 问库存能得到正确回答；任何改库存操作都要您点确认。

---

#### v2.1 处方→出库→打印一条龙

| 项目 | 说明 |
| --- | --- |
| **流程** | 开好处方 → 「生成出库清单」→ 核对 → 确认出库 → 打印 |
| **重要** | 中间任一步取消，不会扣一半库存 |

**验收：** 完整走一遍；取消时库存不变。

---

### 第六期：相似病例参考（v2.2 → v2.3）

| 项目 | 说明 |
| --- | --- |
| **前提** | 系统里已有**足够多**历史病历（建议至少几十份） |
| **医生看到** | 写病历时侧边出现 3 条相似旧病例（**无姓名电话**） |
| **重要** | 界面写清楚：**仅供参考，不能代替诊断** |
| **不想用** | 忽略侧边栏即可 |

**验收：** 相似结果无患者姓名；不影响正常保存病历。

## 九、常见问题

**Q：必须按 v1.1 → v1.2 → v1.3 顺序吗？**  
A：建议按 ROADMAP 顺序。v1.2 可跳过 v1.1 先做；v1.4 依赖 v1.3；v2.0 建议 v1.3 之后。

**Q：升级失败数据会没吗？**  
A：只要没删 `D:\clinic-data\` 且有备份，可以恢复。升级前备份是硬性习惯。

**Q：开发在自己电脑，诊所用另一台，怎么同步？**  
A：开发机用 Cursor 写好 → 打版本 tag → 整包复制 `D:\clinic\` 到诊所机 → 第四节流程。**不要把开发库覆盖诊所库。**

**Q：AI 功能贵吗/要一直联网吗？**  
A：v1.3/v2.0 用 DeepSeek 要联网、按量计费；v1.1 Whisper、v1.4 OCR 可本地容器，不依赖外网 API（除您选用云端 OCR 外）。
